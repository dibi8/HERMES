### Ví Dụ: Triển Khai Hồi Quy Tuyến Tính

Hãy xem xét một phiên bản đơn giản hóa về cách mã của một ngày điển hình có thể trông như thế nào. Dưới đây là một triển khai cơ bản của Hồi quy Tuyến tính bằng Scikit-Learn, đây là điểm khởi đầu phổ biến trong khóa học.

```python
import numpy as np
from sklearn.linear_model import LinearRegression
import matplotlib.pyplot as plt

# Dữ liệu mẫu
X = np.array([[1], [2], [3], [4], [5]])
y = np.array([2, 4, 5, 4, 5])

# Khởi tạo mô hình
model = LinearRegression()

# Huấn luyện mô hình với dữ liệu
model.fit(X, y)

# Dự đoán
predictions = model.predict(X)

# Trực quan hóa kết quả
plt.scatter(X, y, color='red')
plt.plot(X, predictions, color='blue')
plt.title('Ví dụ Hồi quy Tuyến tính')
plt.xlabel('X')
plt.ylabel('y')
plt.show()

print(f"Hệ số: {model.coef_}")
print(f"Giao điểm: {model.intercept_}")
```

Kịch bản đơn giản này minh họa vòng lặp cốt lõi của học máy: chuẩn bị dữ liệu, huấn luyện mô hình, dự đoán và đánh giá. Khi các ngày trôi qua, độ phức tạp tăng lên đáng kể, giới thiệu các khái niệm như gradient descent, kiểm tra chéo (cross-validation) và tinh chỉnh siêu tham số.

### Hướng Dẫn Cài Đặt Từng Bước

#### 1. Sao Chép Kho Lưu Trữ

Đầu tiên, hãy di chuyển đến thư mục mong muốn và sao chép kho lưu trữ từ GitHub.

```bash
git clone https://github.com/Avik-Jain/100-Days-Of-ML-Code.git
cd 100-Days-Of-ML-Code
```

#### 2. Tạo Môi Trường Ảo

Thật tốt khi cô lập các phụ thuộc dự án của bạn. Bạn có thể sử dụng `venv` hoặc `conda`. Dưới đây là cách thiết lập môi trường ảo bằng `venv` tích hợp sẵn của Python.

Mặc dù kho lưu trữ không luôn bao gồm `requirements.txt`, nhưng các thư viện tiêu chuẩn được sử dụng xuyên suốt khóa học đều đã được biết đến rộng rãi. Bạn có thể cài đặt chúng thủ công.

Hầu hết các bài tập được thực hiện trong Jupyter Notebooks. Bạn có thể khởi chạy máy chủ từ thư mục gốc của dự án.

```bash

```

Điều này sẽ mở một cửa sổ trình duyệt nơi bạn có thể điều hướng đến thư mục `Days` và chọn notebook của ngày cụ thể mà bạn muốn làm việc.

### Docker Cho Khả Năng Tái Lập

Đối với người dùng nâng cao hoặc những người quan tâm đến việc triển khai, tạo hình ảnh Docker dựa trên các yêu cầu của khóa học này là một bài tập có giá trị. Điều này đảm bảo rằng môi trường của bạn có thể di chuyển và tái lập trên các máy khác nhau.

```dockerfile
FROM python:3.9-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

CMD ["jupyter", "notebook", "--ip='*'", "--port=8888", "--no-browser", "--allow-root"]
```

Lưu nội dung này dưới dạng `Dockerfile` và xây dựng nó:

```bash
docker build -t ml-course .
```

Chạy container cho phép bạn truy cập các notebook qua trình duyệt web, cách ly hệ thống chủ của bạn khỏi các phụ thuộc.

## Sử Dụng Nâng Cao: Triển Khai Sản Xuất

Sau khi bạn đã làm chủ các khái niệm cơ bản thông qua 100 ngày, bước tiếp theo là áp dụng những kỹ năng này vào môi trường sản xuất. Phần này phác thảo cách lấy một mô hình được phát triển trong kho lưu trữ và triển khai nó dưới dạng dịch vụ web.

### Tuần Tự Hóa Mô Hình

Sau khi huấn luyện một mô hình, bạn cần lưu nó lại để có thể tải nó sau mà không cần huấn luyện lại. Thư viện `joblib` thường được sử dụng cho mục đích này trong Python.

```python
import joblib
from sklearn.linear_model import LogisticRegression
from sklearn.datasets import load_iris

# Tải dữ liệu và huấn luyện mô hình
data = load_iris()
X, y = data.data, data.target
model = LogisticRegression(max_iter=200)
model.fit(X, y)

# Lưu mô hình
joblib.dump(model, 'trained_model.pkl')
print("Mô hình đã được lưu thành công.")
```

### Tạo API Với Flask

Để làm cho mô hình có thể truy cập được qua các yêu cầu HTTP, bạn có thể gói nó trong một API Flask đơn giản.

```python
from flask import Flask, request, jsonify
import joblib
import numpy as np

app = Flask(__name__)

# Tải mô hình đã huấn luyện
model = joblib.load('trained_model.pkl')

@app.route('/predict', methods=['POST'])
def predict():
    try:
        # Lấy dữ liệu JSON từ yêu cầu
        data = request.get_json(force=True)
        features = np.array(data['features']).reshape(1, -1)
        
        # Dự đoán
        prediction = model.predict(features)
        
        return jsonify({'prediction': int(prediction[0])})
    except Exception as e:
        return jsonify({'error': str(e)}), 400

if __name__ == '__main__':
    app.run(debug=True)
```

### Pipeline CI/CD

Tích hợp các pipeline tích hợp liên tục và triển khai liên tục là rất quan trọng để duy trì chất lượng mã. Sử dụng GitHub Actions, bạn có thể tự động chạy các bài kiểm tra bất cứ khi nào bạn đẩy mã mới.

```yaml
name: ML Code Tests
on: [push]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Set up Python
        uses: actions/setup-python@v2
        with:
          python-version: '3.9'
      - name: Install dependencies
        run: |
          pip install -r requirements.txt
      - name: Run tests
        run: |
          pytest tests/
```

Bằng cách làm theo các bước này, bạn chuyển từ người học sang người thực hành, có khả năng xây dựng và triển khai các giải pháp AI thế giới thực.
