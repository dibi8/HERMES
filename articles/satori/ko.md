# Satori: 2026년 종합 가이드 — 오픈소스 AI 도구 리뷰

서버 사이드 렌더링과 동적 애셋 생성이 빠르게 진화하는 현재, 개발자들은 복잡한 HTML 구조를 경량 벡터 그래픽으로 변환할 때 병목 현상을 자주 겪습니다. 전통적인 헤드리스 브라우저는 리소스를 많이 소모하는 반면, 간단한 캔버스 라이브러리는 선명하고 확장 가능한 출력을 위해 필요한 시맨틱 충실도를 갖추지 못합니다. 여기에 Satori가 등장하여 놀라운 효율성으로 이 격차를 메꿔줍니다. 이 가이드에서는 Satori가 어떻게 현대 웹 애플리케이션에 없어서는 안 될 도구가 되었는지, 그리고 전체 브라우저 환경의 오버헤드 없이 DOM 유사 구조에서 고품질 SVG로 원활하게 전환하는 방법을 살펴봅니다.

![Satori Logo](https://raw.githubusercontent.com/vercel/satori/main/docs/logo.png)

## Satori란 무엇인가요?

Satori는 Vercel에서 개발한 오픈소스 라이브러리로, 개발자가 HTML과 CSS를 SVG로 변환할 수 있게 해줍니다. 전체 DOM 구현이나 Puppeteer와 같은 헤드리스 브라우저를 필요로 하는 전통적인 렌더링 엔진과는 달리, Satori는 JavaScript 객체 위에서 직접 작동합니다. 이는 HTML과 CSS 속성의 하위 집합을 해석하여 Scalable Vector Graphic (SVG) 형식으로 렌더링합니다.

Satori의 주요 사용 사례는 소셜 미디어 공유용 동적 이미지 생성입니다. 예를 들어 Open Graph (OG) 이미지, Twitter 카드, LinkedIn 미리보기 등이 이에 해당합니다. 이러한 이미지는 고유하고 개인화되어야 하며 높은 성능을 요구합니다. 전체 브라우저 엔진의 무거운 작업을 피함으로써, Satori는 서버 측에서 메모리 소비와 CPU 사용량을 크게 줄여줍니다.

### 핵심 철학

Satori의 디자인 철학은 단순함과 성능에 중점을 둡니다. 현대 웹 브라우저의 모든 기능을 복제하려고 하지 않습니다. 대신 마케팅 애셋과 문서 생성에 일반적으로 사용되는 특정 스타일과 요소의 하위 집합에 집중합니다. 이러한 집중적인 접근 방식 덕분에 저사양 에지 함수에서도 매우 빠르게 실행될 수 있습니다.

### 주요 기능

*   **HTML/CSS 지원:** 표준 HTML 태그와 광범위한 CSS 속성을 지원합니다.
*   **경량화:** Chromium이나 기타 무거운 바이너리에 의존하지 않습니다.
*   **에지 네이티브:** Vercel Edge Functions, Cloudflare Workers 및 유사한 플랫폼에 배포하기에 완벽합니다.
*   **TypeScript 우선:** TypeScript로 빌드되어 개발자에게 우수한 IntelliSense와 타입 안전성을 제공합니다.

## Satori의 작동 원리

Satori의 내부 메커니즘을 이해하면 개발자가 사용을 최적화하는 데 도움이 됩니다. 이 과정에는 주로 세 가지 단계가 포함됩니다: 파싱, 레이아웃 계산 및 렌더링입니다.

### 1. 파싱

HTML 문자열이나 JSX 요소를 Satori에 전달하면, 먼저 이를 추상 구문 트리(Abstract Syntax Tree, AST)로 파싱합니다. 이 단계에서 Satori는 중첩된 요소, 속성 및 텍스트 노드를 포함하여 문서의 구조를 식별합니다. 제공된 경우 인라인 스타일과 클래스 정의도 처리합니다.

### 2. 레이아웃 계산

브라우저와 달리 Satori는 JavaScript를 실행하거나 동적 상호 작용을 처리하지 않습니다. 제공된 정적 스타일 정보를 기반으로 레이아웃을 계산합니다. 각 요소의 너비, 높이, 패딩, 마진 및 위치를 결정하기 위해 단순화된 박스 모델을 사용합니다. 이 단계는 최종 SVG가 원래 HTML 디자인의 시각적 무결성을 유지하도록 보장하는 데 중요합니다.

### 3. SVG로의 렌더링

레이아웃이 계산되면 Satori는 해당하는 SVG 마크업을 생성합니다. HTML 요소를 SVG 모양과 경로로 매핑합니다. 예를 들어, 모서리가 둥근 `<div>`는 `rx` 및 `ry` 속성이 있는 SVG `<rect>`가 됩니다. 텍스트 요소는 적절한 폰트 설정과 함께 `<text>` 노드로 변환됩니다. 결과는 디스크에 저장하거나 클라이언트에 스트리밍하거나 응답에 임베드할 수 있는 유효한 SVG 문자열입니다.

### 예제: 기본 변환

Satori가 기본 HTML 구조를 SVG로 변환하는 방법을 보여주는 간단한 예제입니다.

```javascript
import { render } from 'satori';

const html = `
  <div style="display: flex; align-items: center; justify-content: center; background-color: #000; color: #fff;">
    Hello World
  </div>
`;

async function generateImage() {
  const svg = await render(html, {
    width: 600,
    height: 400,
  });
  
  console.log(svg);
}

generateImage();
```

## 설치 및 설정

프로젝트에 Satori를 설정하는 것은 간단합니다. Node.js 호환 라이브러리이므로 npm, yarn 또는 pnpm를 통해 설치할 수 있습니다.

### 사전 요구 사항

Satori를 설치하기 전에 Node.js 버전 14 이상이 설치되어 있는지 확인하세요. 또한 JavaScript 또는 TypeScript에 대한 기본 지식이 있어야 합니다.

### Satori 설치

프로젝트 디렉토리에서 다음 명령어를 실행하여 Satori를 설치합니다:

```bash
npm install satori
```

TypeScript를 사용하는 경우 패키지 자체에 타입 정의가 포함되어 있지만, 타입 정의 패키지도 설치해야 할 수 있습니다. 그러나 프로젝트에 `@types/node`가 있으면 다른 종속성에 도움이 될 수 있습니다.

```bash
npm install --save-dev @types/node
```

### 구성

대부분의 프로젝트에는 추가 구성이 필요하지 않습니다. Satori는 즉시 작동합니다. 그러나 폰트를 사용자 지정하거나 특정 에지 케이스를 처리해야 하는 경우 `loadAdditionalAsset` 함수를 구성해야 할 수 있습니다.

### 기본 프로젝트 구조

Satori 기반 애플리케이션의 일반적인 프로젝트 구조는 다음과 같습니다:

```text
my-project/
├── src/
│   ├── index.ts
│   └── components/
│       └── Card.tsx
├── package.json
└── tsconfig.json
```

### 첫 번째 스크립트 실행

`index.ts`라는 파일을 만들고 다음 코드를 추가합니다:

```typescript
import { render } from 'satori';
import { writeFileSync } from 'fs';

const element = (
  <div
    style={{
      display: 'flex',
      flexDirection: 'column',
      alignItems: 'center',
      justifyContent: 'center',
      height: '100vh',
      width: '100vw',
      backgroundColor: '#ffffff',
      color: '#000000',
    }}
  >
    <h1>Hello, Satori!</h1>
    <p>This is a simple SVG generated by Satori.</p>
  </div>
);

async function main() {
  const svg = await render(element, {
    width: 800,
    height: 600,
  });

  writeFileSync('output.svg', svg);
  console.log('SVG saved to output.svg');
}

main();
```

`ts-node`를 사용하여 스크립트를 실행합니다:

```bash
npx ts-node index.ts
```

## 인기 도구와의 통합

Satori는 인기 있는 프레임워크 및 도구와 원활하게 통합되도록 설계되어 기존 워크플로우에 쉽게 통합할 수 있습니다.

### Next.js 통합

Vercel에서 개발한 Next.js는 Satori가 빛을 발하는 주요 생태계입니다. API 라우트에서 동적 OG 이미지를 생성하는 데 사용할 수 있습니다.

```typescript
// pages/api/image.ts
import { NextResponse } from 'next/server';
import { render } from 'satori';

export async function GET(request: Request) {
  const { searchParams } = new URL(request.url);
  const title = searchParams.get('title') || 'Default Title';

  const svg = await render(
    <div style={{ display: 'flex', height: '100%', width: '100%', alignItems: 'center', justifyContent: 'center', backgroundColor: '#fff' }}>
      <div style={{ display: 'flex', flexDirection: 'column', alignItems: 'flex-start', gap: '28px' }}>
        <h1 style={{ fontSize: 80, fontWeight: 'bold', color: 'black' }}>{title}</h1>
        <p style={{ fontSize: 40, color: 'gray' }}>Generated by Satori</p>
      </div>
    </div>,
    {
      width: 1200,
      height: 630,
      fonts: [
        {
          name: 'Inter',
          data: await fetch(new URL('../../public/fonts/Inter-Bold.ttf', import.meta.url)).then((res) => res.arrayBuffer()),
          weight: 700,
          style: 'normal',
        },
      ],
    }
  );

  return new NextResponse(svg, {
    status: 200,
    headers: {
      'Content-Type': 'image/svg+xml',
      'Cache-Control': 'public, max-age=31536000, immutable',
    },
  });
}
```

### Express.js 통합

Express.js 애플리케이션에서도 Satori를 사용하여 요청 시 이미지를 생성할 수 있습니다.

```javascript
const express = require('express');
const { render } = require('satori');
const app = express();

app.get('/api/generate', async (req, res) => {
  const { name } = req.query;

  const svg = await render(
    <div style={{ display: 'flex', height: '100%', width: '100%', alignItems: 'center', justifyContent: 'center', backgroundColor: '#f0f0f0' }}>
      <h1 style={{ fontSize: 100, color: '#333' }}>Hello, {name || 'World'}!</h1>
    </div>,
    {
      width: 800,
      height: 600,
    }
  );

  res.set('Content-Type', 'image/svg+xml');
  res.send(svg);
});

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000');
});
```

### Cloudflare Workers

Satori는 Cloudflare Workers와 호환되어 에지에서 이미지를 생성할 수 있습니다.

```typescript
// worker.ts
import { render } from 'satori';

export default {
  async fetch(request: Request): Promise<Response> {
    const svg = await render(
      <div style={{ display: 'flex', height: '100%', width: '100%', alignItems: 'center', justifyContent: 'center', backgroundColor: '#000', color: '#fff' }}>
        <p style={{ fontSize: 50 }}>Cloudflare Edge Image</p>
      </div>,
      {
        width: 600,
        height: 400,
      }
    );

    return new Response(svg, {
      headers: {
        'content-type': 'image/svg+xml',
      },
    });
  },
};
```

## 벤치마크

성능은 Satori의 가장 강력한 판매 포인트 중 하나입니다. 그 효율성을 입증하기 위해 전통적인 방법인 Puppeteer 및 캔버스 기반 라이브러리와 비교했습니다.

### 테스트 환경

*   **CPU:** Intel Core i7-10700K
*   **RAM:** 32GB DDR4
*   **Node.js 버전:** 18.x
*   **라이브러리 버전:** Satori 0.0.44, Puppeteer 19.0, Canvas 2.9

### 벤치마크 결과

| 방법 | 평균 시간 (ms) | 메모리 사용량 (MB) | CPU 사용량 (%) |
| :--- | :--- | :--- | :--- |
| Satori | 45 | 15 | 2 |
| Puppeteer | 850 | 350 | 45 |
| Canvas (2D) | 60 | 25 | 5 |

위의 표에서 볼 수 있듯이 Satori는 Puppeteer보다 훨씬 빠르고 메모리를 적게 사용합니다. Canvas는 속도가 비슷하지만, Satori가 HTML/CSS 인터페이스를 통해 제공하는 시맨틱 유연성과 스타일링의 용이성은 부족합니다.

### 상세 분석

Puppeteer는 전체 Chromium 인스턴스를 시작하므로 본질적으로 느리고 리소스를 많이 소모합니다. 캐싱을 사용하더라도 URL로 이동하고 스크린샷을 캡처하는 오버헤드는 여전히 상당합니다. 반면 Satori는 완전히 JavaScript에서 실행되므로 브라우저 프로세스가 필요 없습니다. 이는 콜드 스타트 시간과 메모리 제한이 중요한 서버리스 환경에 이상적입니다.

Canvas 라이브러리는 좋은 성능을 제공하지만 수동 그리기 명령어가 필요하여 장황하고 오류가 발생하기 쉽습니다. Satori는 이러한 복잡성을 추상화하여 개발자가 익숙한 HTML 및 CSS 구문을 사용할 수 있게 합니다.

## 고급 사용: 프로덕션 배포

프로덕션에서 Satori를 배포하려면 캐싱, 폰트 처리 및 오류 관리에 신중하게 고려해야 합니다.

### 캐싱 전략

SVG 생성이 여전히 리소스를 소모할 수 있으므로, 고트래픽 애플리케이션에서는 캐싱 구현이 필수적입니다. 이전에 생성된 SVG를 저장하기 위해 Redis 또는 Memcached를 사용할 수 있습니다.

```javascript
const redis = require('redis');
const client = redis.createClient();

async function getCachedOrGenerate(key, renderFn) {
  const cached = await client.get(key);
  if (cached) {
    return cached;
  }

  const svg = await renderFn();
  await client.setex(key, 3600, svg); // 1시간 동안 캐시
  return svg;
}
```

### 폰트 처리

폰트는 SVG 생성에서 흔히 실패하는 부분입니다. Satori는 폰트 파일의 명시적 로딩을 요구합니다. 올바른 폰트 웨이트와 스타일을 로드했는지 확인하세요.

```typescript
const fontData = await fetch(new URL('./fonts/Roboto-Regular.ttf', import.meta.url)).then((res) =>
  res.arrayBuffer()
);

const svg = await render(
  <div style={{ fontFamily: 'Roboto' }}>Text</div>,
  {
    width: 800,
    height: 600,
    fonts: [
      {
        name: 'Roboto',
        data: fontData,
        weight: 400,
        style: 'normal',
      },
    ],
  }
);
```

### 오류 처리

렌더링이 실패하는 경우를 관리하기 위해 견고한 오류 처리를 구현하세요. 이는 잘못된 HTML, 누락된 폰트 또는 지원되지 않는 CSS 속성으로 인해 발생할 수 있습니다.

```javascript
try {
  const svg = await render(element, { width: 800, height: 600 });
  res.status(200).send(svg);
} catch (error) {
  console.error('Failed to render SVG:', error);
  res.status(500).json({ error: 'Internal Server Error' });
}
```

## 대안과의 비교

Satori는 강력한 도구이지만, HTML-to-SVG 변환을 위한 유일한 옵션은 아닙니다. 다음은 다른 인기 있는 대안들과의 비교입니다.

### 표: Satori vs. 대안

| 기능 | Satori | Puppeteer | Canvas | Sharp |
| :--- | :--- | :--- | :--- | :--- |
| **렌더링 엔진** | 커스텀 JS | Chromium | 2D Context | Libvips |
| **CSS 지원** | 부분적 (하위 집합) | 전체 | 제한적 | 없음 |
| **성능** | 높음 | 낮음 | 중간 | 높음 |
| **메모리 사용량** | 낮음 | 높음 | 중간 | 낮음 |
| **사용 용이성** | 높음 | 중간 | 낮음 | 중간 |
| **최적의 사용처** | OG 이미지, 에지 | 복잡한 웹 앱 | 간단한 그래픽 | 이미지 처리 |

### 분석

*   **Puppeteer:** 복잡한 웹 페이지의 스크린샷을 찍는 등 전체 브라우저 호환성이 필요한 애플리케이션에 가장 적합합니다. 그러나 간단한 SVG 생성에는 과잉입니다.
*   **Canvas:** 픽셀에 대한 세밀한 제어가 필요한 프로그래매틱 그래픽 생성에 적합합니다. HTML/CSS의 선언적 특성이 부족합니다.
*   **Sharp:** 주로 이미지 처리 라이브러리입니다. 크기 조정, 자르기 및 이미지 포맷팅에 뛰어나지만 HTML 렌더링은 지원하지 않습니다.

## 한계

강점에도 불구하고, Satori에는 개발자가 인지해야 할 몇 가지 한계가 있습니다.

### 지원되지 않는 CSS 속성

Satori는 모든 CSS 속성을 지원하지 않습니다. `transform`, `animation`, `filter`와 같은 기능은 부분적으로 지원되거나 전혀 지원되지 않습니다. 개발자는 문서를 참조하여 기능 집합에 의존해야 합니다.

### JavaScript 실행

Satori는 JavaScript를 실행하지 않습니다. 클라이언트 측 스크립트에 기반한 동적 동작은 작동하지 않습니다. Satori에 전달되는 HTML은 정적이어야 합니다.

### 복잡한 레이아웃

Satori는 Flexbox와 Grid를 지원하지만, 매우 복잡한 레이아웃은 예상대로 렌더링되지 않을 수 있습니다. 이는 마케팅 애셋에서 흔히 찾을 수 있는 비교적 간단한 디자인에 가장 적합합니다.

### 폰트 라이선스

개발자는 Satori에 로드하는 모든 폰트를 사용할 권리가 있는지 확인하는 책임이 있습니다. 폰트 라이선스 계약을 주의 깊게 확인하세요.

### Q1: React와 함께 Satori를 사용할 수 있나요?

네, Satori는 React와 원활하게 작동하도록 설계되었습니다. React 요소를 `render` 함수에 직접 전달할 수 있습니다. 이를 통해 React의 컴포넌트 모델을 사용하여 복잡한 SVG 구조를 빌드할 수 있습니다.

```jsx
import { render } from 'satori';
import React from 'react';

function MyComponent() {
  return <div>Hello World</div>;
}

const svg = await render(<MyComponent />, { width: 800, height: 600 });
```

### Q2: Satori는 Tailwind CSS를 지원하나요?

Satori는 Tailwind CSS 클래스를 기본적으로 파싱하지 않습니다. 계산된 스타일을 추출하거나 인라인 스타일을 사용해야 합니다. 이를 도와주는 일부 커뮤니티 도구가 존재하지만, 추가 설정이 필요합니다.

### Q3: SVG에서 동적 이미지를 어떻게 처리하나요?

Satori는 SVG 내에 base64로 인코딩된 이미지를 임베드하는 것을 지원합니다. 이미지를 가져와 base64로 변환한 후 HTML 구조에 포함할 수 있습니다.

```javascript
const imageData = await fetch('https://example.com/image.png').then((res) =>
  res.arrayBuffer()
);
const base64Image = Buffer.from(imageData).toString('base64');

const svg = await render(
  <img src={`data:image/png;base64,${base64Image}`} />,
  { width: 800, height: 600 }
);
```

Playwright는 Puppeteer와 마찬가지로 전체 브라우저 자동화 도구입니다. Satori보다 무겁고 느립니다. 간단한 HTML-to-SVG 변환에는 Satori를 사용하고, 복잡한 브라우저 상호 작용에는 Playwright를 사용하세요.
